## Next.js landing page

<Card
  title="Open the Next.js example"
  icon="code"
  href="https://github.com/Open-Chat-Widget/openchatwidget/tree/main/examples/nextjs-landing-page"
  horizontal
>
  Use the same setup as the repo's landing page: a Next.js App Router app with a serverless `POST /api/chat` route and the widget mounted on the page.
</Card>

- Uses `examples/nextjs-landing-page`
- Good reference for a full-site integration
- Streams replies from a Next.js route handler
- Keeps the widget mounted in a client wrapper

## React + Express example

<Card
  title="Open the React + Express example"
  icon="server"
  href="https://github.com/Open-Chat-Widget/openchatwidget/tree/main/examples/basic-react-express-app"
  horizontal
>
  Use this example when you want the frontend and agent server separated.
</Card>

- Uses `examples/basic-react-express-app`
- Frontend runs on `http://localhost:5173`
- Express agent runs on `http://localhost:8787/api/chat`
- Good reference for local development and custom backends
