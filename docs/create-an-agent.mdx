title: "Create an agent"
description: "Build a minimal Vercel AI SDK backend and expose a streaming endpoint for Open Chat Widget."
---

Once the widget is mounted, the only required backend contract is a streaming chat endpoint. The example below uses Express, the Vercel AI SDK, and the helper exports from `@openchatwidget/sdk`.

<Info>
  **Environment variables**:
  - Set `OPENAI_API_KEY` before starting the server.
  - The example below exposes `POST /api/chat` on `http://localhost:8787`.
</Info>

<Steps>
  <Step title="Create the server">

```ts
import "dotenv/config";
import cors from "cors";
import express from "express";
import {
  convertToModelMessages,
  createOpenAI,
  streamText,
  type UIMessage,
} from "@openchatwidget/sdk";

const app = express();

app.use(cors());
app.use(express.json());

app.post("/api/chat", async (request, response) => {
  const { messages } = request.body as { messages: UIMessage[] };

  const openai = createOpenAI({
    apiKey: process.env.OPENAI_API_KEY,
  });

  const result = streamText({
    model: openai("gpt-4o-mini"),
    system: "You are the Open Chat Widget example assistant. Keep answers concise and useful.",
    messages: await convertToModelMessages(messages),
  });

  result.pipeUIMessageStreamToResponse(response);
});

app.listen(8787, () => {
  console.log("Express agent listening on http://localhost:8787");
});
```

  </Step>
  <Step title="Connect the widget">

```tsx
<OpenChatWidget url="http://localhost:8787/api/chat" />
```

  </Step>
  <Step title="Test the full loop">
    Start your frontend and backend. Open the site, send a message, and confirm that the widget streams responses from your agent.
  </Step>
</Steps>

## Why this endpoint works

- `messages` come from the widget in AI SDK UI message format.
- `convertToModelMessages()` converts those messages into model-ready input.
- `streamText()` generates a streaming response.
- `pipeUIMessageStreamToResponse()` sends the stream back in the format the widget expects.
